# Introducing Limbo: A Complete Rewrite of SQLite in Rust

**Authors:** Pekka Enberg and Glauber Costa
**Published:** December 10, 2024
**Source:** https://turso.tech/blog/introducing-limbo-a-complete-rewrite-of-sqlite-in-rust

## Background: From Fork to Rewrite

Two years before Limbo, Turso forked SQLite to create libSQL — an open contribution project that reached 12,000+ GitHub stars, 85 contributors, and added features like native replication and vector search.

The fork had tradeoffs: SQLite's test suite is proprietary, making large changes risky. SQLite's C implementation also created obstacles for safely evolving the codebase. When adding indexed vector search, the team had to choose between unintuitive syntax or invasive bytecode changes. This prompted the question: what would a complete rewrite from scratch in Rust entail?

## Deterministic Simulation Testing (DST)

The key reliability strategy behind Limbo is DST, embedded from day one.

> "Deterministic Simulation Testing is a paradigm made famous by the folks at TigerBeetle."

With DST built in from the beginning, developers can:
- Simulate unlikely scenarios easily
- Test years of execution with different event orderings
- Reproduce any found issue 100% reliably

Turso also partnered with **Antithesis** — an autonomous testing platform that runs code in a deterministic hypervisor, injecting faults while continuously verifying system properties. Antithesis identified issues in Limbo's io_uring implementation during partial write operations — an extremely rare condition that internal testing frameworks would likely miss.

Beyond simulation, the team regularly fuzzes inputs and verifies that generated bytecode matches between Limbo and SQLite.

## Technical Highlights

- **Fully asynchronous I/O** from inception (unlike SQLite's synchronous interface), using io_uring on Linux.
- **Designed for WASM** from the ground up — includes VFS compatibility with tools like Drizzle without modifications.
- **Performance**: early benchmarks show `SELECT * FROM users LIMIT 1` at 506ns vs SQLite's 620ns (20% improvement) on M2 MacBook Air, with default settings.

## Relationship to libSQL

Limbo is not positioned as a competitor to libSQL. If successful, the Limbo codebase would simply become libSQL, released under the MIT license.

---

**Key takeaway for rewrites:** Starting a rewrite with Deterministic Simulation Testing built-in from the get-go — rather than bolting it on later — is what makes Turso's approach different from past failed rewrites. Testing confidence is the foundation that makes large changes safe.
